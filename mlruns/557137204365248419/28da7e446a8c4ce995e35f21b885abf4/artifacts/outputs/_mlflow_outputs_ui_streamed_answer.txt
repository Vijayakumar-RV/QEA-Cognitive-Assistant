### US-09-TC001 — Verify successful user login with valid credentials
**Type:** Manual  |  **Priority:** Critical  
**Tags:** login, positive, authentication

**Description**  
Test case to ensure a returning user can successfully log in to their account using valid credentials and is redirected to the My Account dashboard.

**Preconditions**
- A registered user account exists with valid login credentials (username and password).

**Test Steps**
1. {'step_number': 1, 'action': 'Navigate to the Automation Test Store Home Page.', 'expected_outcome': 'The Home Page is displayed with the URL: https://www.automationteststore.com/'}
2. {'step_number': 2, 'action': "Click on the 'Login or Register' link in the header.", 'expected_outcome': 'The My Account Dashboard page (user_not_logged_in variant) is displayed, showing the login form.'}
3. {'step_number': 3, 'action': "Enter a valid username into the 'Login Name' input field.", 'expected_outcome': "The username is populated in the 'Login Name' field."}
4. {'step_number': 4, 'action': "Enter a valid password into the 'Password' input field.", 'expected_outcome': "The password is populated in the 'Password' field."}
5. {'step_number': 5, 'action': "Click on the 'Login Button'.", 'expected_outcome': "The user is successfully logged in and redirected to the My Account Dashboard page (user_logged_in_dashboard variant), displaying options such as 'Edit your account information', 'Change your password', 'Modify your address book entries', 'View your order history', 'Downloads', 'Subscribe / unsubscribe to newsletter', and 'Logoff'."}

**Expected Result**
> The user is successfully logged in and redirected to the My Account Dashboard, confirming access to account management features.

### US-09-TC002 — Verify unsuccessful user login with invalid credentials
**Type:** Manual  |  **Priority:** High  
**Tags:** login, negative, authentication, error_handling

**Description**  
Test case to ensure the system handles invalid login attempts gracefully by displaying an error message and preventing access to the account dashboard.

**Preconditions**
- None

**Test Steps**
1. {'step_number': 1, 'action': 'Navigate to the Automation Test Store Home Page.', 'expected_outcome': 'The Home Page is displayed with the URL: https://www.automationteststore.com/'}
2. {'step_number': 2, 'action': "Click on the 'Login or Register' link in the header.", 'expected_outcome': 'The My Account Dashboard page (user_not_logged_in variant) is displayed, showing the login form.'}
3. {'step_number': 3, 'action': "Enter an invalid username (e.g., 'invaliduser') into the 'Login Name' input field.", 'expected_outcome': "The invalid username is populated in the 'Login Name' field."}
4. {'step_number': 4, 'action': "Enter an invalid password (e.g., 'invalidpass') into the 'Password' input field.", 'expected_outcome': "The invalid password is populated in the 'Password' field."}
5. {'step_number': 5, 'action': "Click on the 'Login Button'.", 'expected_outcome': 'An error message indicating invalid credentials is displayed on the page, and the user remains on the My Account Dashboard page (user_not_logged_in variant).'}

**Expected Result**
> An appropriate error message for invalid login credentials is displayed, and the user is not redirected to the My Account Dashboard.
